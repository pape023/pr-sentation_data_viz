01 — Objectif

Pourquoi ce projet ?

La gestion scolaire de l'ENSAE reposait sur des fichiers Excel partagés, sans contrôle d'accès ni intégrité des données. Ce projet replace tout par une application web robuste.

warning

Problème initial

Fichiers Excel partagés, risques de corruption, aucun contrôle d'accès, pas d'historique, impossible de gérer plusieurs utilisateurs simultanément.

rocket_launch

Solution déployée

Application web Dash centralisée avec base SQL relationnelle, authentification multi-rôles, génération PDF automatique et import Excel intelligent.

verified

Cahier des charges

Tous les modules du projet Dataviz2 sont couverts et enrichis de fonctionnalités bonus : planning collaboratif, statistiques, explorateur BDD.

02 — Architecture

Stack & Structure

Architecture MVC légère : Dash gère les vues et callbacks, SQLAlchemy l'accès aux données, les utilitaires assurent les services transverses (PDF, email, migration).

terminal

Python 3.11 · Dash 2.17

Framework réactif Plotly. Routing multi-pages natif (use_pages=True). Sessions dcc.Store. Composants Bootstrap.

database

SQLAlchemy 2.0 ORM

SQLite en dev (zéro config), PostgreSQL 15 en production. Switch via variable DATABASE_URL sans changer une ligne de code.

lock

bcrypt · Sessions

Hachage adaptatif des mots de passe. Sessions vérifiées à chaque chargement de page via require_auth().

picture_as_pdf

ReportLab 4.2

Bulletins PDF avec charte ENSAE, tableaux, calculs automatiques de moyennes et de rangs par classe.

table_chart

pandas · openpyxl

Import multi-feuilles, export templates pré-remplis, validation des données avant insertion SQL.

rocket_launch

Gunicorn · Render.com

Serveur WSGI de production. Déploiement continu GitHub → Render. HTTPS automatique Let's Encrypt.

04 — Modules applicatifs

10 pages, 1 mission chacune

Chaque fichier dans pages/ est une route Dash indépendante avec ses callbacks, sa logique métier et son contrôle d'accès.

login

login.py — /

Layout split. Vérifie email + bcrypt, crée la session dcc.Store avec user_id, role, nom, prenom. Redirect vers /dashboard.

dashboard

dashboard.py — /dashboard

KPIs dynamiques selon le rôle. Admin → stats globales. Resp. filière → sa filière. Délégué → sa classe. Étudiant → ses indicateurs personnels.

menu_book

cours.py — /cours

CRUD UE et Modules via modals. Barre de progression en temps réel (heures effectuées / volume horaire prévu). Champ email enseignant pour notifications planning.

calendar_today

seances.py — /seances

Création séance avec module, date, créneau, thème. Appel numérique : checklist générée dynamiquement — coché = absent. Historique filtrable.

people

etudiants.py — /etudiants

Liste + recherche. Fiche : assiduité, moyenne générale, absences détaillées. Workflow notes : template → saisie → upload → moyennes auto.

picture_as_pdf

bulletins.py — /bulletins

PDF individuel ou lot classe. Notes, coefficients, moyenne pondérée, rang, appréciations. Charte ENSAE via ReportLab.

event_note

planning.py — /planning

Calendrier interactif. Workflow : Délégué soumet → email → Resp. filière valide/rejette → email retour.

bar_chart

statistiques.py — /statistiques

Graphiques Plotly : distribution notes, assiduité par classe, progression horaire. Scope auto selon rôle, filtre classe.

manage_accounts

admin.py — /admin

CRUD utilisateurs et rôles. Import Excel 7 feuilles avec rapport détaillé. Gestion filières/classes. Désignation délégués.

database

db.py — /db Admin only

16 tables avec compteurs temps réel. Tableau paginé, recherche textuelle, édition inline via modal, suppression avec confirmation.

07 — Déploiement

De GitHub à la prod

Déploiement continu sur Render.com depuis la branche main. PostgreSQL managé, HTTPS automatique, variables d'environnement pour les secrets.

computerInstallation locale

bash — setup local
# 1. Cloner
git clone https://github.com/papamagattediop/SGA_ENSAE && cd SGA_ENSAE

# 2. Environnement virtuel
python -m venv .venv && source .venv/bin/activate

# 3. Dépendances
pip install -r requirements.txt

# 4. Configurer les secrets
cp .env.example .env   # puis éditer MAIL_ADDRESS, MAIL_PASSWORD, DATABASE_URL

# 5. Initialiser la BDD
python recreate_db.py

# 6. Lancer → http://127.0.0.1:8050
python app.py

08 — Guide de test

10 scénarios à valider

Parcours complet dans l'ordre — certaines étapes créent les données nécessaires aux suivantes.

1
Authentification & Session
Prérequis
  • Se connecter admin@ensae.sn / admin123 → redirection /dashboard
  • Navbar affiche tous les menus admin
  • Tester email incorrect → message d'erreur affiché
  • Déconnexion via menu avatar → retour à /
2
Migration Excel → SQL
Admin
  • Générer le template : python utils/migration.py --template
  • Remplir feuilles Filieres, Classes, Etudiants
  • Upload drag & drop → rapport s'affiche (✅ importés / ⚠️ ignorés / ❌ erreurs)
  • Ré-upload même fichier → aucun doublon, avertissements uniquement
  • Log visible dans Historique des imports
3
Gestion Cours & UE
Admin / Resp. filière
  • Créer UE UE-STAT, sélectionner classe et période
  • Créer Module rattaché : 30h, coef 2, email enseignant
  • Barre de progression affiche 0% (aucune séance encore)
  • Modifier le module → changements persistés après rechargement
4
Séances & Appel numérique
Multi-rôles
  • Créer séance : module, date, 08h–10h, thème "Introduction"
  • Checklist de tous les étudiants de la classe générée automatiquement
  • Cocher 2 étudiants absents → valider
  • Historique affiche la séance · barre progression Cours +2h/30h
  • Taux d'assiduité des absents diminue dans leurs fiches
5
Workflow Notes
Admin / Resp. filière
  • Télécharger template notes pour une classe (Excel avec colonnes Note1 / Note2)
  • Remplir les notes de 5 étudiants sur 2 modules
  • Upload → moyennes calculées automatiquement, visibles dans les fiches
  • Moyenne générale et rang mis à jour dans la classe
6
Bulletins PDF
Multi-rôles
  • Générer bulletin individuel pour un étudiant avec notes
  • PDF contient : nom, classe, tableau notes, coefficients, moyenne, rang
  • Générer en lot pour toute une classe
  • PDF multi-pages, un bulletin par étudiant, charte ENSAE
7
Workflow Planning collaboratif
Multi-rôles requis
  • Se connecter en resp_classe → créer planning semaine suivante avec 3 séances
  • Soumettre → statut passe à soumis
  • Reconnecter en resp_filiere → planning visible
  • Valider → statut validé, email envoyé au délégué
  • Rejeter avec commentaire → commentaire obligatoire validé
8
Statistiques interactives
Admin / Resp. filière
  • Graphiques Plotly affichés : distribution notes, assiduité, progression
  • Filtre par classe → mise à jour dynamique
  • Se connecter en resp_filiere → scope limité à sa filière
9
Explorateur BDD
Admin uniquement
  • 16 tables listées avec compteurs temps réel
  • Tableau paginé avec recherche textuelle fonctionnelle
  • Modifier un champ → persistance confirmée après rechargement
10
Sécurité & Contrôle d'accès
Critique
  • Se connecter en étudiant → tenter /admin manuellement
  • Redirection vers login — accès refusé
  • Même résultat pour /db et /statistiques
  • Se déconnecter → tenter /dashboard sans session
  • Redirection login — session expirée détectée
